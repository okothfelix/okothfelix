Bee Multiscent

  • Company Name

    Bee Multiscent Limited Company

  • Project Duration

    1 year

Project Description

Problem Statement

Bee Multiscent is a smart aromatheraputic assistant that contains several luxury scents and medicinal scents that can be diffused based on the users current mood. There was need to come up with such a device because in the maret there is no diffuser where the user could be able to mix different flavors at a go and get a custom flavor. The Bee Multiscent diffuser is capable of handling this and more. I joined the team as technical lead where I was in charge of designing the application and web version of the diffuser that would enable the user to remotely change the scent they desire, set the timer on how long the diffuser should be on and report on the scent levels to check if they needed refill.

Key Steps

The device had three user interfaces for development, the mobile application, web portal and remote control for offline access.
i) The mobile application.
Through the mobile application the users were able to connect to their own devices and regulate the scents remotely. The app also provided notifications on scent levels and recommendations on when to purchase the scents once they were finished.
ii) The Web Application Portal
The portal was mainly for hotels that had installed the diffuser to check on the levels of the scents and to pre-condition the rooms to smell a particular scent before the arrival of the room owner. Through this portal the administrators would turn the diffusers on/off and set them to a given schedule remotely. They would also be able to get scent level data ad notifications on when to change the scent bottles.
iii) The remote control
The remote control served as an alternative by enabling access to the diffuser when it was not accessible by both the mobile application for the home-owners / the web-portal for the hotels.

Deliverables

The application development was successful and I was able to integrate all the three user interface to work seamlessly with the diffuser.

Key Learning Points

Data acquisition and processing: I learnt to collect data from sensors, process it, and potentially store it for further analysis or visualization.
Communication protocols: I implemented communication protocols like MQTT which is crucial. MQTT enables lightweight and efficient data exchange between devices and servers.

Conclusion

Having worked with the Bee Multiscent Startup I learned a lot in terms of developing iot systems that interact with different software programs at a go and how to optimize them for a fast response time. Overall, developing an IoT application with Python provided a hands-on experience in hardware interaction, data processing, communication protocols, and backend development.
