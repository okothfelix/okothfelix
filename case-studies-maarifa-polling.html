Project Description

Problem Statement

During the 2022 general elections I was actively involved with the communication team of my area women representative and we faced a challenge of getting voters feedback fast on the items in our agenda. I came up with Maarifa Polling a crowd-sourcing application that enabled us to gather responses on surveys from both closed and oen audiences. Some of the key specifications of the platform were that the administrator should be able to load the surveys and view analytics on how each survey is performing based on the different target audience.

Key Steps

The platform needed to be accessed by both users with smart-phones and feature phones, therefore there was need to develop a USSD application to mae this functionality possible. The administrators of the platform also required a web portal to ba able to load all the surveys and to view analytics of how the voters were responding.
i) The Unstructured Supplementary Service Data (USSD) application.
Through this application the voters would log in their responses. The ussd application was developed in python and flask framework.
ii) The Web Application Portal
Through this portal the administrators would load the surveys and view analytics of the voters responses.
iii) A Short Code Service
Through this service, respondents would be notified of any changes in the candidates schedule and would also be awarded for the paid surveys that were added to the platform.

Deliverables

The application development was successful and I was able to help the communication team with the challenge of bulk messaging, data processing and planning of the entire campaign communication strategy. Eventually our candidate won the seat.

Key Learning Points

This project highlighted the vital role ussd applications have in collection of information especially through surveys. The application was really effective and cheap compared to if I had developed an application and since most people are familiar with ussd codes it was easy to reach a big number of participants.
Speed was a factor in this project and everything had to be done fast from the development of the application to using it in production. To reduce development time I used a skeleton key I had developed both the ussd and web application.
The blueprint is available in my git-hub account here.

Conclusion

Having blueprints/skeleton keys as a developer is important and can save a lot of time and reduce burnout especially if the timelines are intense.
